# 🤝 How to Contribute

Thank you for your interest in contributing! We welcome contributions of all kinds, including:
- 🚀 New features
- 🐛 Bug fixes
- 📝 Documentation improvements
- ✅ Tests and refactoring

# 🛠️ Development Setup
1. Fork this repository
2. Clone your fork:
```bash
 
git clone https://github.com/YOUR_USERNAME/this-repo.git
cd this-repo

```
3. Install dependencies:

```bash

pip install -r requirements.txt

```

# 🔧 Coding Guidelines
- Follow PEP8 for Python code style
- Use clear and concise commit messages (e.g. fix: handle missing states in rollout)
- Format your code using black or flake8 if availab

# 📦 Submitting a Pull Request
1. Create a new branch:

```bash

git checkout -b feature/your-feature-name

```
2. Make your changes and commit:

```bash

git add .
git commit -m "feat: add curiosity reward visualization"

```
3. Push to your fork and create a Pull Request on GitHub

✅ Please ensure:
- Your code runs without errors
- You describe what your PR does and why it’s valuable
- Related issues (if any) are mentioned using Closes #xx

# 💬 Opening an Issue
If you found a bug or have a suggestion:
- Provide environment info (Python version, OS)
- Provide reproduction steps or error logs
- Suggest possible fixes if you have any
